# Intro - 데이터 사고

0. 직장인에게 데이터가 필요한 이유
+ 무엇인가를 주장하는 데 근거가 필요하기 때문에 데이터를 조사하고 수집하고 가공하고 분석하고 시각화하는 것.
+ 상사의 so what? 논리에 대응
+ 자료에서 사실, 사실에서 판단, 판단에서 주장으로 대응(반대는 why so? process)

1. 우리에게 필요한 데이터 역량
+ 자료: 내가 주장하고자 하는 것에 알맞은 자료 모으기
+ 사실: 자료에서 쓸모 있는 사실 꺼내기
+ 판단: 올바른 판단을 내리는 능력 - 똑같은 사실을 가지고도 전혀 다르게 판단 할 수 있음
+ 주장: 최선의 주장 만들기(다양한 주장만들기(안) + 그 안을 올리기(기안))

2. 확산과 수렴의 사고
+ 확산: 문제 해결을 위한 여러가지 아이디어 제안 - 현재 좋은 아이디어가 있더라도 더 좋은 아이이어 도출<br/> 회사가 공모전을 하는 이유임
+ 수렴: 아이디어를 많이 낸 다음에 좋은 아이디어 몇 개를 선정하는 것.
+ 자료, 사실, 판단, 주장 모든 과정에서 확산과 수렴 반복 - 4 다이아몬드 모델

3. 확산과 수렴의 사고에서 지켜야 할 원칙
+ 같은 자료에서 완전히 다른 주장이 나올 수 있다는 것 기억
+ 어떤 사실을 이야기할 때는 반드시 자료로 검증
+ 판단할 때는 기준이 있어야함 ex. 80점을 받았으면 잘 한건지? 평균이 60점이었다면 잘 한 것
+ 실행할 수 없는 주장은 주장이 아님.

4. 사실
+ 쓸데없는 사실: 차트를 보면 금방 알 수 있는 것을 굳이 말이나 글로 설명 -> 그냥 차트에 적기<br/> ex. A제품 매출 증가
+ 쓸모 있는 사실: 판단과 주장으로 이어짐<br/> ex. A제품 매출이 이례적으로 증가, B제품 품질 문제로 2년째 매출 하락, C제품 판매량이 펴오와 달리 감소
+ 즉, 쓸모 있는 사실은 이례, 예외, 특이, 패턴, 예상과 다른 결과, 인과가 분명한 것, 행동이 필요한 것, 대책이 필요한 것들임

4.1. 사실과 진실 비교하기
+ 사실: 실제로 있었던 일, 현재에 있는 일
<br/> 사실을 보여주는 척 하면서 조작할 수도 있음 ex. 6월 매출이 전월 대비 반토막(5월과 6월만 비교해서 보여줌)
+ 진실: 거짓이 없는 사실<br/> 특정 기간만 비교 X, 기간을 다양하게 비교  <br/> 월단위로만 봤다면 분기, 반기, 연으로 혹은 주별, 일별, 시간대별로 
<br/> 항목 비교 - A제품이 이례적으로 증가했다고 하기 전에 A제품과 유사한 B제품, 또 다른 모든 제품의 평균이나 합과 비교
<br/> 구성 비교 - 원가가 올라 매출이 늘었는지, 많이 팔려서 매출액이 늘었는지 분명히 구분

5. 판단
+ 가치 판단: 비교나 준거 대상이 있어야 함 ex. 과거 실적, 업계 연봉인상률 <br/> 가치판단 예시. 3월3주 서울 사무실 매매가는 전주보다 0.01% *소폭* 상승
+ 상황 판단: 판단하는 사람의 상황에 따라 다 다르게 판단 <br/> 상황판단 예시. 서울 사무실 매매가 폭등, *대책 필요*
+ 이번 달 매출이 25억으로 반토막(가치판단) -> 매출 급락에 적극대응(상황판단1) or 매출 급락 상황을 주시(상황판단2) or 매출 급락 원인 파악(상황판단3)

# Intro - 데이터 기획

1. 데이터 분류와 구분
+ 분석을 하려면 분류부터 ex. 돈을 쓴 용도- 식비, 생봘비, 문화비...
+ 분류 기준만 달리해도 다른 데이터가 나옴 ex. 돈을 쓴 용도별, 장소별, 지불방법별 등등
+ 분류: 상위 개념에서 하위 개념으로 구체화하는 것 -> 조사<br/> 고객 만족도 -> 품질, 가격, 직원친절도
+ 구분: 하위 개념을 가지고 상위 개념으로 추상화하는 것 -> 보고<br/> 고양이, 소, 토끼 -> 포유류
+ 상위 개념을 하위 개념으로 나눌 때 하위 개념 사이에 중복과 누락이 있어선 안된다.

2. 시계열 차트 패턴으로 기획하기(시간 순서)
+ 차트는 데이터 분석의 새로운 시작 -> 비즈니스 자체엔 일정한 패턴이 존재함
+ 계절성: 일반적으로 많은 비즈니스는 계절성을 띰<br/> 현재 제품이 여름에만 잘 팔린다면 호빵처럼 겨울에 잘 팔릴 신제품을 기획해야할 것
+ 비교성: 패턴은 다른 것과 비교했을 때 드러나는 경우도 많음 <br/> A폰의 판매량이 늘어날 때 B폰의 판매량이 줄어듬.

3. 분산 차트로 프레임워크 만들기(산점도)
+ 패턴이 있는 경우: 영업사원의 고객 방문과 매출의 관계 + 추세선+ R-제곱값
+ 패턴이 없는 경우: 4개의 사분면으로 나누기 -> 2Y2 매트릭스

4. 2Y2 매트릭스로 기획하기
+ 각 사분면 정의하고, 사분면간 이동을 만들기

4. 이상 패턴 찾아 기획하기
+ 정상 패턴보다는 이상 패턴을 잘 찾아야함 -> 엑셀로 추세선 그려보기(R-제곱값 표시)

5. 데이터로 5Whys 사용하기 - Why?Why?Why?Why?Why?
+ 예시. 기념관의 대리석이 빨리 부식됨 
<br/> 첫 번쨰 질문. 왜 기념관 대리석 지붕이 빨리 부식되는가? - 대리석 지붕을 세제로 자주 씻어서
<br/> 두 번째 질문. 왜 세제로 대리석을 씻는가? - 비둘기 배설물이 많아서
<br/> 세 번째 질문. 왜 비둘기가 많은가? - 비둘기 먹이인 거미가 많아서
<br/> 네 번째 질문. 왜 거미가 많은가? - 거미의 먹이인 나방이 많아서
<br/> 다섯 번째 질물. 왜 나방이 많은가? - 해가 완전히 지기 전에 전등을 일찍 켜서
+ 꼭 다섯 번이 아니어도됨. 다섯 번 정도의 질문에서 근본 원인을 찾는 기법

6. 원인과 이유
+ 원인: 사물이나 현상에 사용 ex. 감기의 원인? 바이러스
+ 이유: 사람에게 사용 ex. 감기에 걸린 이유? 추운데 밖에서 돌아다녀서
+ 많은 사람들이 5Whys에 원인과 이유를 섞어 사용하는 실수를함 ->원인 5Whys와 이유 5Whys 나눠서 사용
+ 예시

      <원인 5Whys> - 사물이나 현상
      1. 왜 로그인 기능이 작동하지 않는가?<br/>아이디가 13글자 이상 입력되면 에러가 발생
      2. 왜 서버에서 에러가 발생하는가?<br/>서버 프로그램에서는 아이디를 12글자만 받을 수 있기 때문이다.
      3. 서버 프로그램에서 왜 아이드를 12글자만 받을 수 있는가?<br/>데이터베이스에서 아이디를 12글자로 제한했기 때문이다.
      4. 데이터베이스에서 왜 아이디를 12글자로 제한하는가?<br/>데이터베이스의 아이디 기본 설정이 12글자이기 때문이다.
      5. 데이터베이스ㅢ 아이디 기본 설정이 왜 12글자인가?<br/>아이디가 12글자 이내일 때 데이터베이스의 속도가 가장 높기 때문이다.
      
      <이유 5Whys> - 사람
      1. 왜 로그인 기능이 작동하지 않는가?<br/>사용자가 아이디 칸에 13글자 이상 입력해서
      2. 사용자는 왜 12글자 이상 입력하는가?<br/>프런트 개발자가 13글자 이상을 입력하게 허용했기 때문이다.
      3. 프런트 개발자는 왜 13글자 이상을 입력하게 허용했는가?<br/>프런트 개발자가 아이디 글자 수 제한을 몰랐기 때문이다.
      4. 프런트 개발자는 왜 아이디 글자 수 제한을 몰랐는가?<br/>서버 개발자가 프런트 개발자에게 아이디 수 제한을 알리지 않았기 때문이다.
      5. 왜 서버 개발자는 프런트 개발자에게 아이디 수 제한을 알리지 않았는가?<br/> 서버 개발자는 프런트 개발자가 누구인지 몰랐기 때문이다.

+ 원인을 해결하는 단기대책과 장기대책, 이유를 해결하는 단기대책과 장기대책이 모두 나옴 -> '입체적인 대책'
+ 원인을 해결하는 단기대책: 데이터베이스에서 글자 수 제한 없애기
+ 원인을 해결하는 장기대책: 데이터베이스 속도를 측정해 최적 아이디 글자 수 알아내기
+ 이유를 해결하는 단기대책: 프런트 개발자에게 아이디 글자 수 제한 알려주기
+ 이유를 해결하는 장기대책: 서버 개발자와 프런트 개발자를 한 팀으로 묶거나 정기 회의 하기

# Intro - 대시보드로 보고하기
+ 대시보드: 행동지표와 목표 지표를 같이 보여줌으로써 운전자가 지금 무슨 행동을 하는지, 그것이 목표에 적합한지, 목표를 얼마나 당성하고 있는지, 앞으로 어떻게 해야 목표를 달성할 수 있는지 알려줌.

1. 데이터로 만드는 목표와 지표
+ 목표는 지향하는 가치, 명확히 수치로 나올 수 있어야 함
 
2. 잘못된 목표의 예시
<br/>ex.신규 시장에 진출한다 -> 올해 안에 신규 시장에 진출한다. -> 신규 시장에 진출하여 1년 안에 시장 점유율 20%를 달성한다. -> 1년 안에 시장 점유율 20%를 달성하여 해외 시장 개척의 디딤돌을 만든다.
+ 신규 시장에 진출한다: 목표가 아닌 전략적 목적지에 불과
+ 올해 안에: 올해 라는 기준 척도가 존재, 하지만 월 해야하는지 행동 목표가 없음
+ 즉, 목표는 인과관계가 명시되어야 함
+ 해외 사장 개척의 디딤돌을 만든다: 얼핏 보면 행동 목표 같지만, 기준과 척도를 주지 못함.

3. 올바른 목표의 예시
<br/> A 제품의 포털 배너 광고를 10억 원어치 집행해서 *1안에 신규 방문자 100만 명, 구매 20만 명을 달성한다.*
<br/> 영업 팀이 매월 신규 고객을 방문하여 *신규 고객 주문 비율을 10% 높인다.*
<br/> 모든 임직원이 멘토링에 참여하여 *소통 지수를 10% 높인다.*
+ 행동 지표 + 결과 지표

4. 대시보드의 명확한 설계 원칙
+ 대시보드를 사용하는 목적을 명확히 설명 - 자동차 대시보드에 자동차 운전과 관련 없는 것 빼기 ex. 바퀴 색깔.
+ 대시보드의 지표는 분석 대상이 아님 - 보는 즉시 이해 가능?
+ 지표는 적을수록 좋고, 중요한 것은 크게 보여야 함 ex. 자동차 대시보드에서 속도를 가장 크게 보여줌
+ 지표에는 기준과 수준을 포함 ex. 시속 200km(기준)가 넘으면 위험 표시
<br/> 기준을 초과하거나 미달한 정도- 201km와 250km는 또 다름
